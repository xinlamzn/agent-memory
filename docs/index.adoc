== Three Memory Types, One Graph

Most agent memory systems give you a flat context window or a simple vector store.
Neo4j Agent Memory gives your agent three distinct memory layers -- all connected in a single knowledge graph.

++++
<div class="memory-types-grid">
  <div class="memory-card">
    <div class="memory-card-icon short-term"><i class="fa-solid fa-comments"></i></div>
    <h3>Short-Term Memory</h3>
    <p class="memory-card-subtitle">Conversations &amp; experiences</p>
    <p>Full conversation history with sequential message chains, session management, metadata-based search, and conversation summaries.</p>
    <ul class="memory-card-features">
      <li>Semantic message search</li>
      <li>Session scoping &amp; listing</li>
      <li>Metadata filters</li>
      <li>LLM-powered summaries</li>
    </ul>
  </div>
  <div class="memory-card">
    <div class="memory-card-icon long-term"><i class="fa-solid fa-brain"></i></div>
    <h3>Long-Term Memory</h3>
    <p class="memory-card-subtitle">Facts, entities &amp; preferences</p>
    <p>A knowledge graph of entities (people, places, orgs), preferences, facts, and relationships &mdash; built automatically from conversations.</p>
    <ul class="memory-card-features">
      <li>POLE+O entity classification</li>
      <li>Entity deduplication</li>
      <li>Temporal fact validity</li>
      <li>Geospatial queries</li>
    </ul>
  </div>
  <div class="memory-card">
    <div class="memory-card-icon reasoning"><i class="fa-solid fa-route"></i></div>
    <h3>Reasoning Memory</h3>
    <p class="memory-card-subtitle">Tool usage &amp; reasoning traces</p>
    <p>Records how the agent solved problems &mdash; every thought, tool call, and outcome &mdash; so it can learn from past reasoning patterns.</p>
    <ul class="memory-card-features">
      <li>Trace similarity search</li>
      <li>Tool call statistics</li>
      <li>Message-linked traces</li>
      <li>Streaming trace recording</li>
    </ul>
  </div>
</div>

[source,python]
----
async with MemoryClient(settings) as memory:
    # Short-term: store conversations
    await memory.short_term.add_message(session_id, "user", "Find Italian restaurants near me")

    # Long-term: build a knowledge graph
    await memory.long_term.add_entity("La Trattoria", "ORGANIZATION", subtype="COMPANY")

    # Reasoning: record how the agent solved problems
    trace = await memory.reasoning.start_trace(session_id, task="Restaurant search")
    await memory.reasoning.record_tool_call(step.id, "search_api", {"query": "Italian"}, results)
    await memory.reasoning.complete_trace(trace.id, outcome="Recommended La Trattoria", success=True)

    # Get unified context for LLM prompts
    context = await memory.get_context("restaurant recommendation", session_id=session_id)
----

== What Makes This Different

++++
<div class="differentiators-grid">
  <div class="diff-card">
    <div class="diff-icon"><i class="fa-solid fa-diagram-project"></i></div>
    <h4>Graph-Native, Not Bolted On</h4>
    <p>Built on Neo4j from the ground up. Entities, messages, traces, and preferences are all <strong>nodes and relationships</strong> &mdash; not rows in a table or chunks in a vector store. Query across memory types with a single traversal.</p>
  </div>
  <div class="diff-card">
    <div class="diff-icon"><i class="fa-solid fa-magnifying-glass"></i></div>
    <h4>Vector + Graph + Spatial in One</h4>
    <p>Semantic similarity search, graph traversal, and geospatial queries all in <strong>one database</strong>. No separate vector DB, no Redis, no Postgres. Find entities by meaning, by relationship, or by location.</p>
  </div>
  <div class="diff-card">
    <div class="diff-icon"><i class="fa-solid fa-gears"></i></div>
    <h4>Multi-Stage Entity Extraction</h4>
    <p>Combine <strong>spaCy, GLiNER2, GLiREL, and LLM extractors</strong> in a configurable pipeline. 8 domain schemas (podcast, news, medical, legal...). Streaming extraction for 100K+ token documents.</p>
  </div>
  <div class="diff-card">
    <div class="diff-icon"><i class="fa-solid fa-layer-group"></i></div>
    <h4>Three Memory Layers, Connected</h4>
    <p>Short-term conversations inform long-term entity extraction. Reasoning traces link back to triggering messages. The agent understands <strong>what happened, what it knows, and how it solved things</strong>.</p>
  </div>
  <div class="diff-card">
    <div class="diff-icon"><i class="fa-solid fa-globe"></i></div>
    <h4>Wikipedia Enrichment &amp; Geocoding</h4>
    <p>Entities are automatically enriched with <strong>Wikipedia descriptions, images, and Wikidata IDs</strong>. Location entities get coordinates for geospatial queries. All in the background.</p>
  </div>
  <div class="diff-card">
    <div class="diff-icon"><i class="fa-solid fa-plug"></i></div>
    <h4>Framework Agnostic</h4>
    <p>Works with <strong>LangChain, PydanticAI, LlamaIndex, and CrewAI</strong>. Or use the client directly. Your agent framework, your choice &mdash; the memory layer stays the same.</p>
  </div>
</div>

== Quick Start

[source,bash]
----
pip install neo4j-agent-memory[openai]
----

[source,python]
----
import asyncio
from pydantic import SecretStr
from neo4j_agent_memory import MemoryClient, MemorySettings

async def main():
    settings = MemorySettings(
        neo4j={"uri": "bolt://localhost:7687", "password": SecretStr("password")}
    )

    async with MemoryClient(settings) as memory:
        # Store a conversation
        await memory.short_term.add_message("user-123", "user", "I love Italian food!")

        # Build knowledge automatically
        await memory.long_term.add_preference("food", "Loves Italian cuisine", importance=0.9)

        # Get combined context for your LLM
        context = await memory.get_context("restaurant recommendation", session_id="user-123")
        print(context)

asyncio.run(main())
----
