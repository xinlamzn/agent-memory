= Entity Extraction Pipeline
:toc: left
:toclevels: 3
:sectanchors:
:sectlinks:
:icons: font
:source-highlighter: highlight.js
:experimental:

Neo4j Agent Memory includes a powerful, configurable entity extraction pipeline that automatically identifies and classifies entities from text using the xref:poleo-model.adoc[POLE+O data model].

== Overview

The extraction pipeline supports three extraction engines that can be used individually or combined:

[cols="1,1,2,1"]
|===
|Engine |Type |Best For |Speed

|**spaCy**
|Statistical NER
|Common entities (people, organizations, locations)
|Fast (~1ms)

|**GLiNER2**
|Zero-shot NER
|Domain-specific entities with schemas
|Medium (~50ms)

|**LLM**
|Language Model
|Complex extraction, relations, preferences
|Slow (~500ms)
|===

== Pipeline Architecture

[source]
----
                    Input Text
                        │
                        ▼
            ┌───────────────────────┐
            │   Stage 1: spaCy      │  Fast statistical NER
            │   (en_core_web_sm)    │  → PERSON, ORG, GPE, etc.
            └───────────┬───────────┘
                        │
                        ▼
            ┌───────────────────────┐
            │   Stage 2: GLiNER2    │  Zero-shot NER with
            │   (gliner_medium)     │  domain schemas
            └───────────┬───────────┘
                        │
                        ▼
            ┌───────────────────────┐
            │   Stage 3: LLM        │  Complex extraction,
            │   (gpt-4o-mini)       │  relations, preferences
            └───────────┬───────────┘
                        │
                        ▼
            ┌───────────────────────┐
            │   Stopword Filtering  │  Remove invalid entities
            └───────────┬───────────┘
                        │
                        ▼
            ┌───────────────────────┐
            │    Merge Results      │  Combine using strategy:
            │    (CONFIDENCE)       │  union/intersection/
            └───────────────────────┘  confidence/cascade
                        │
                        ▼
              ExtractionResult
              - entities[]
              - relations[]
----


[[gliner2]]
== GLiNER2: Zero-Shot Named Entity Recognition

GLiNER2 is a lightweight model that can extract any entity type without retraining. Unlike traditional NER models that only recognize pre-defined types, GLiNER2 accepts a list of entity types at inference time.

=== Key Features

[horizontal]
Zero-shot extraction:: Define custom entity types without training
Entity type descriptions:: Provide descriptions to improve accuracy
Domain schemas:: Pre-defined schemas for common use cases (<<domain-schemas>>)
Lightweight:: Runs on CPU, ~500MB model size

=== Model Selection

[cols="2,1,2"]
|===
|Model |Size |Notes

|`gliner-community/gliner_small-v2.5`
|~200MB
|Fastest, less accurate

|`gliner-community/gliner_medium-v2.5`
|~500MB
|Balanced (default)

|`gliner-community/gliner_large-v2.5`
|~1GB
|Most accurate, slower
|===

=== Checking GLiNER Availability

GLiNER is an optional dependency. Check availability before use:

[source,python]
----
from neo4j_agent_memory.extraction import is_gliner_available

if is_gliner_available():
    from neo4j_agent_memory.extraction import GLiNEREntityExtractor
    extractor = GLiNEREntityExtractor.for_schema("podcast")
else:
    print("GLiNER not installed. Install with: pip install neo4j-agent-memory[gliner]")
----


[[domain-schemas]]
== Domain Schemas

Domain schemas provide entity type definitions optimized for specific content types. Each schema includes:

. **Entity types** -- What to extract (e.g., "person", "company")
. **Descriptions** -- Natural language descriptions that help GLiNER2 understand what to look for

IMPORTANT: Domain schemas are specific to GLiNER2. spaCy uses fixed trained types, and the LLM extractor uses prompt-based extraction without detailed descriptions.

=== Available Schemas

[cols="1,2,2"]
|===
|Schema |Entity Types |Best For

|`poleo`
|person, organization, location, event, object
|General/investigations

|`podcast`
|person, company, product, concept, book, role, metric, technology
|Podcast transcripts

|`news`
|person, organization, location, event, date
|News articles

|`scientific`
|author, institution, method, dataset, metric, concept, tool
|Research papers

|`business`
|company, person, product, industry, financial_metric, location
|Business content

|`entertainment`
|actor, director, film, tv_show, character, award, studio, genre
|Movies/TV

|`medical`
|disease, drug, symptom, procedure, body_part, gene, organism
|Healthcare

|`legal`
|case, person, organization, law, court, date, monetary_amount
|Legal documents
|===

=== Using Domain Schemas

[source,python]
----
from neo4j_agent_memory.extraction import (
    GLiNEREntityExtractor,
    get_schema,
    list_schemas,
)

# List all available schemas
print(list_schemas())
# ['poleo', 'podcast', 'news', 'scientific', 'business', 'entertainment', 'medical', 'legal']

# Get a specific schema
schema = get_schema("podcast")
print(schema.entity_types)
# {'person': 'A person mentioned in the podcast...', 'company': '...', ...}

# Create extractor with schema (recommended)
extractor = GLiNEREntityExtractor.for_schema("podcast", threshold=0.45)

# Extract entities
result = await extractor.extract(
    "Lenny talked with Brian Chesky about Airbnb's growth strategy"
)

# Filter and display results
for entity in result.filter_invalid_entities().entities:
    print(f"{entity.name}: {entity.type} (confidence: {entity.confidence:.2f})")
# Brian Chesky: PERSON (confidence: 0.92)
# Airbnb: ORGANIZATION (confidence: 0.89)
----

=== Using Schemas with the Pipeline

When using `ExtractorBuilder`, the schema applies to the GLiNER stage:

from neo4j_agent_memory.extraction import ExtractorBuilder

=== Creating Custom Schemas

Create custom schemas for your specific domain:

[source,python]
----
from neo4j_agent_memory.extraction.domain_schemas import DomainSchema

# Define a custom schema
real_estate_schema = DomainSchema(
    name="real_estate",
    entity_types={
        "property": "A real estate property, building, or land parcel",
        "agent": "A real estate agent or broker representing buyers or sellers",
        "buyer": "A property buyer, purchaser, or potential customer",
        "seller": "A property seller, owner, or listing party",
        "price": "A property price, asking price, sale price, or valuation",
        "neighborhood": "A neighborhood, district, area, or community",
        "feature": "A property feature, amenity, or characteristic",
    },
)

# Create extractor with custom schema
extractor = GLiNEREntityExtractor(schema=real_estate_schema, threshold=0.5)

# Extract entities
result = await extractor.extract(
    "John Smith is selling his 3-bedroom home in Westwood for $1.2M. "
    "The property features a pool and updated kitchen."
)
----

== Multi-Stage Pipeline

=== Configuration

[source,python]
----
from neo4j_agent_memory import MemorySettings
from neo4j_agent_memory.config.settings import ExtractionConfig

settings = MemorySettings(
    extraction=ExtractionConfig(
        # Enable/disable stages
        enable_spacy=True,
        enable_gliner=True,
        enable_llm_fallback=True,

        # GLiNER settings
        gliner_model="gliner-community/gliner_medium-v2.5",
        gliner_threshold=0.5,
        gliner_schema="podcast",

        # Merge strategy
        merge_strategy="confidence",

        # Continue to next stage if current returns empty
        fallback_on_empty=True,
    )
)
----

=== Merge Strategies

[cols="1,3"]
|===
|Strategy |Behavior

|`union`
|Keep all unique entities from all stages

|`intersection`
|Only keep entities found by multiple extractors (higher confidence)

|`confidence`
|Keep the highest-confidence version of each unique entity (default)

|`cascade`
|Use first stage results, fill gaps with subsequent stages

|`first_success`
|Stop after the first stage that returns results
|===


== Stopword Filtering

After extraction, entities are automatically filtered to remove noise:

[source,python]
----
from neo4j_agent_memory.extraction import is_valid_entity_name, ENTITY_STOPWORDS

# Check if a name is valid
is_valid_entity_name("John Smith")  # True
is_valid_entity_name("they")        # False (pronoun)
is_valid_entity_name("123")         # False (numeric)
is_valid_entity_name("a")           # False (too short)

# Filter results automatically
result = await extractor.extract(text)
filtered = result.filter_invalid_entities()
----

.Filtered Categories
****
* **Pronouns**: "they", "them", "you", "me", "it", "we"
* **Common verbs**: "is", "are", "was", "have", "do"
* **Articles**: "a", "an", "the"
* **Numeric values**: "10", "123.45", "50%"
* **Conversation artifacts**: "um", "uh", "hmm"
****

=== Tips

. **Use the right schema** -- Domain-specific schemas significantly improve accuracy
. **Adjust threshold** -- Lower threshold = more entities (but more noise)
. **Disable unused stages** -- Skip spaCy or LLM if not needed

. **Batch processing** -- Use `extract_batch()` for multiple texts (see <<batch-extraction>>)


[[batch-extraction]]
== Batch Extraction

For processing multiple texts efficiently, use the `extract_batch()` method which provides:

* Parallel processing with configurable concurrency
* Progress tracking via callbacks
* Aggregate statistics and error handling
* Memory-efficient batching

=== Pipeline Batch Extraction

[source,python]
----
from neo4j_agent_memory.extraction import (
    ExtractionPipeline,
    BatchExtractionResult,
)

pipeline = ExtractionPipeline(stages=[spacy_extractor, gliner_extractor])

# Process multiple texts with progress tracking
texts = ["Text 1...", "Text 2...", "Text 3...", ...]

def on_progress(completed: int, total: int) -> None:
    print(f"Progress: {completed}/{total} ({100*completed/total:.0f}%)")

result: BatchExtractionResult = await pipeline.extract_batch(
    texts,
    batch_size=10,           # Texts per memory batch
    max_concurrency=5,       # Parallel extractions
    on_progress=on_progress, # Progress callback
    fail_fast=False,         # Continue on errors (default)
)

# Aggregate statistics
print(f"Processed: {result.total_items}")
print(f"Successful: {result.successful_items}")
print(f"Failed: {result.failed_items}")
print(f"Success rate: {result.success_rate:.1%}")
print(f"Total entities: {result.total_entities}")
print(f"Total relations: {result.total_relations}")
print(f"Duration: {result.total_duration_ms:.0f}ms")

# Access all extracted entities
all_entities = result.get_all_entities()

# Access individual results (maintains input order)
for item in result.results:
    if item.success:
        print(f"Text {item.index}: {item.result.entity_count} entities in {item.duration_ms:.1f}ms")
    else:
        print(f"Text {item.index}: FAILED - {item.error}")

# Get just the errors
for index, error_msg in result.get_errors():
    print(f"Item {index} failed: {error_msg}")
----

=== GLiNER Batch Extraction (GPU-Optimized)

GLiNER supports native batch inference which is significantly faster on GPU:

[source,python]
----
from neo4j_agent_memory.extraction import GLiNEREntityExtractor

# Create extractor with GPU
extractor = GLiNEREntityExtractor.for_schema(
    "podcast",
    device="cuda",  # or "mps" for Apple Silicon
)

# Native batch extraction uses GLiNER's batch_predict_entities
results = await extractor.extract_batch(
    texts,
    batch_size=32,  # Larger batches for GPU efficiency
    on_progress=on_progress,
)

# results is a list of ExtractionResult objects
for i, result in enumerate(results):
    print(f"Text {i}: {result.entity_count} entities")
----

=== Batch Parameters

[cols="1,1,3"]
|===
|Parameter |Default |Description

|`batch_size`
|10
|Number of texts processed before yielding to event loop (memory management)

|`max_concurrency`
|5
|Maximum number of parallel extractions (pipeline only)

|`on_progress`
|None
|Callback function `(completed, total) -> None`

|`fail_fast`
|False
|If True, stop on first error; if False, collect errors and continue
|===

=== BatchExtractionResult Properties

[cols="1,3"]
|===
|Property |Description

|`total_items`
|Total number of texts processed

|`successful_items`
|Number of successfully processed texts

|`failed_items`
|Number of failed texts

|`success_rate`
|Success rate as a fraction (0.0-1.0)

|`total_entities`
|Sum of entities across all successful extractions

|`total_relations`
|Sum of relations across all successful extractions

|`total_duration_ms`
|Total processing time in milliseconds

|`results`
|List of `BatchItemResult` objects (one per text)
|===

=== Error Handling

[source,python]
----
# Continue on errors (default)
result = await pipeline.extract_batch(texts, fail_fast=False)
if result.failed_items > 0:
    print(f"Warning: {result.failed_items} texts failed")
    for index, error in result.get_errors():
        print(f"  Text {index}: {error}")

# Stop on first error
try:
    result = await pipeline.extract_batch(texts, fail_fast=True)
except Exception as e:
    print(f"Extraction failed: {e}")
----


[[streaming-extraction]]
== Streaming Extraction for Long Documents

For processing very long documents (>100K tokens), use `StreamingExtractor` to:

* Process documents chunk by chunk (memory efficient)
* Yield results as each chunk completes (real-time feedback)
* Automatically deduplicate entities across chunks
* Preserve entity positions relative to the original document

=== Basic Usage

[source,python]
----
from neo4j_agent_memory.extraction import (
    StreamingExtractor,
    create_streaming_extractor,
    GLiNEREntityExtractor,
)

# Create base extractor
extractor = GLiNEREntityExtractor.for_schema("podcast")

# Wrap with streaming extractor
streamer = StreamingExtractor(
    extractor,
    chunk_size=4000,         # Characters per chunk
    overlap=200,             # Overlap between chunks
    split_on_sentences=True, # Try to split on sentence boundaries
)

# Or use factory with defaults
streamer = create_streaming_extractor(extractor)
----

=== Streaming Results

Process chunks as they complete:

[source,python]
----
# Stream results chunk by chunk (async generator)
async for chunk_result in streamer.extract_streaming(long_document):
    print(f"Chunk {chunk_result.chunk.index}:")
    print(f"  Characters: {chunk_result.chunk.start_char}-{chunk_result.chunk.end_char}")
    print(f"  Entities: {chunk_result.entity_count}")
    if not chunk_result.success:
        print(f"  Error: {chunk_result.error}")
----

=== Complete Extraction with Deduplication

Get aggregated results with automatic deduplication:

[source,python]
----
result = await streamer.extract(
    long_document,
    deduplicate=True,  # Remove duplicate entities across chunks
    on_progress=lambda done, total: print(f"Progress: {done}/{total}"),
)

print(f"Chunks processed: {result.stats.total_chunks}")
print(f"Raw entities: {result.stats.total_entities}")
print(f"After deduplication: {result.stats.deduplicated_entities}")

# Convert to standard ExtractionResult
extraction_result = result.to_extraction_result(source_text=long_document)
----

=== Token-Based Chunking

Chunk by approximate token count instead of characters:

[source,python]
----
streamer = StreamingExtractor(
    extractor,
    chunk_size=1000,      # Tokens per chunk
    overlap=50,           # Token overlap
    chunk_by_tokens=True, # Use token-based chunking
)
----

=== Chunking Utilities

Use standalone chunking functions for custom processing:

[source,python]
----
from neo4j_agent_memory.extraction import chunk_text_by_chars, chunk_text_by_tokens

# Character-based chunking
chunks = chunk_text_by_chars(text, chunk_size=4000, overlap=200)

# Token-based chunking
chunks = chunk_text_by_tokens(text, chunk_size=1000, overlap=50)

for chunk in chunks:
    print(f"Chunk {chunk.index}:")
    print(f"  Position: {chunk.start_char}-{chunk.end_char}")
    print(f"  Is first: {chunk.is_first}, Is last: {chunk.is_last}")
    print(f"  Approx tokens: {chunk.approx_token_count}")
----

from neo4j_agent_memory.extraction import GLiNERWithRelationsExtractor

# Create combined extractor for POLE+O model

    print(f"  {entity.name}: {entity.type}")

print("Relations:")
for relation in result.relations:

[[entity-deduplication]]
== Entity Deduplication on Ingest

When storing extracted entities to Neo4j via `LongTermMemory`, automatic deduplication can identify and handle duplicate entities using embedding similarity and optional fuzzy string matching.

=== Configuration

[source,python]
----
from neo4j_agent_memory.memory import (
    LongTermMemory,
    DeduplicationConfig,
)

# Configure deduplication thresholds
dedup_config = DeduplicationConfig(
    enabled=True,                    # Enable deduplication (default)
    auto_merge_threshold=0.95,       # Auto-merge if similarity >= 0.95
    flag_threshold=0.85,             # Flag for review if >= 0.85 but < 0.95
    use_fuzzy_matching=True,         # Also use fuzzy string matching
    fuzzy_threshold=0.9,             # Fuzzy match threshold
    max_candidates=10,               # Max candidates to check
    match_same_type_only=True,       # Only match entities of same type
)

# Pass config when creating LongTermMemory
long_term = LongTermMemory(
    client=neo4j_client,
    embedder=embedder,
    deduplication=dedup_config,
)
----

=== Deduplication Behavior

When adding an entity, the system:

1. **Generates embedding** for the new entity name
2. **Searches existing entities** by embedding similarity
3. **Optionally applies fuzzy string matching** to boost confidence
4. **Takes action based on similarity score**:

[cols="1,2,2"]
|===
|Score Range |Action |Result

|>= 0.95 (auto_merge)
|Auto-merge
|Returns existing entity; adds new name as alias

|>= 0.85 (flag) but < 0.95
|Flag for review
|Creates new entity with `SAME_AS` relationship

|< 0.85
|No duplicate
|Creates new entity normally
|===

=== Usage

[source,python]
----
# add_entity now returns (entity, dedup_result) tuple
entity, dedup_result = await long_term.add_entity(
    name="Jon Smith",  # Variant spelling
    entity_type="PERSON",
)

if dedup_result.action == "merged":
    print(f"Auto-merged with {dedup_result.matched_entity_name}")
    # entity is the existing entity
elif dedup_result.action == "flagged":
    print(f"Flagged as potential duplicate of {dedup_result.matched_entity_name}")
    # New entity created with SAME_AS relationship for review
else:
    print("No duplicate found")

# Disable deduplication for specific entity
entity, _ = await long_term.add_entity(
    name="New Entity",
    entity_type="OBJECT",
    deduplicate=False,  # Skip deduplication check
)
----

=== Managing Duplicates

[source,python]
----
# Find pending duplicate reviews
duplicates = await long_term.find_potential_duplicates(limit=100)
for entity1, entity2, confidence in duplicates:
    print(f"{entity1.name} ↔ {entity2.name} ({confidence:.1%})")

# Confirm a duplicate (merge)
await long_term.review_duplicate(
    source_id=entity1.id,
    target_id=entity2.id,
    confirm=True,  # Merge source into target
)

# Reject a duplicate (not the same entity)
await long_term.review_duplicate(
    source_id=entity1.id,
    target_id=entity2.id,
    confirm=False,  # Mark as distinct entities
)

# Get all entities in a SAME_AS cluster
cluster = await long_term.get_same_as_cluster(entity_id)

# Get deduplication statistics
stats = await long_term.get_deduplication_stats()
print(f"Total entities: {stats.total_entities}")
print(f"Merged entities: {stats.merged_entities}")
print(f"Pending reviews: {stats.pending_reviews}")
----

=== SAME_AS Relationship Schema

Flagged duplicates create `SAME_AS` relationships in Neo4j:

[source,cypher]
----
(Entity)-[:SAME_AS {
    confidence: 0.88,
    match_type: "embedding",  // or "fuzzy" or "both"
    status: "pending",        // or "confirmed" or "rejected"
    created_at: datetime()
}]->(Entity)
----

=== Disabling Deduplication

[source,python]
----
# Globally disable deduplication
dedup_config = DeduplicationConfig(enabled=False)

# Or per-entity basis
entity, _ = await long_term.add_entity(
    name="Entity Name",
    entity_type="PERSON",
    deduplicate=False,
)
----


== Location Geocoding

Extracted LOCATION entities can be automatically geocoded to add latitude/longitude coordinates, enabling geospatial queries like "find nearby locations" or "locations within a bounding box".

=== Enabling Geocoding

[source,python]
----
from neo4j_agent_memory import (
    MemorySettings,
    GeocodingConfig,
    GeocodingProvider,
)

# Enable geocoding with Nominatim (free, rate-limited)
settings = MemorySettings(
    geocoding=GeocodingConfig(
        enabled=True,
        provider=GeocodingProvider.NOMINATIM,
    )
)

# Or with Google Maps (higher accuracy, requires API key)
settings = MemorySettings(
    geocoding=GeocodingConfig(
        enabled=True,
        provider=GeocodingProvider.GOOGLE,
        api_key=SecretStr("your-google-api-key"),
    )
)
----

=== Automatic Geocoding During Extraction

When geocoding is enabled and a LOCATION entity is extracted from text, it is automatically geocoded:

[source,python]
----
async with MemoryClient(settings) as memory:
    # Entity extraction from messages automatically geocodes LOCATION entities
    await memory.short_term.add_message(
        session_id="user-123",
        role="user",
        content="I visited the Eiffel Tower in Paris last week",
        extract_entities=True,
    )
    # "Eiffel Tower" and "Paris" LOCATION entities are geocoded automatically
----

=== Batch Geocoding

For existing LOCATION entities without coordinates:

[source,python]
----
# Batch geocode all locations without coordinates
stats = await memory.long_term.geocode_locations(
    batch_size=50,
    skip_existing=True,  # Skip locations that already have coordinates
)
print(f"Geocoded {stats['geocoded']} of {stats['processed']} locations")
----

=== Geospatial Queries

Once locations are geocoded, you can run geospatial queries:

[source,python]
----
# Find locations within 10km of a point
nearby = await memory.long_term.search_locations_near(
    latitude=48.8584,
    longitude=2.2945,
    radius_km=10.0,
    limit=20,
)

# Find locations within a bounding box
in_area = await memory.long_term.search_locations_in_bounds(
    min_lat=48.80,
    max_lat=48.90,
    min_lon=2.20,
    max_lon=2.40,
)

# Get coordinates for a specific entity
coords = await memory.long_term.get_entity_coordinates(entity_id)
if coords:
    lat, lon = coords
    print(f"Location: {lat}, {lon}")
----
