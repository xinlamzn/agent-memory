= Background Entity Enrichment
:toc: left
:toclevels: 3
:sectanchors:
:icons: font
:source-highlighter: highlight.js

Automatically enrich extracted entities with additional data from external knowledge sources like Wikipedia and Diffbot.

== Overview

The enrichment system fetches additional context about entities from external services after they are extracted and stored. This is useful for:

* Adding detailed descriptions to entities
* Linking entities to external knowledge bases (Wikidata, Diffbot KG)
* Getting images, URLs, and structured metadata
* Finding related entities

Enrichment is **non-blocking** - entities are stored immediately, and enrichment happens asynchronously in the background.

== Architecture

[source,text]
----
┌─────────────────────────────────────────────────────────────┐
│                      MemoryClient                           │
│  - Creates enrichment service from EnrichmentConfig         │
│  - Manages background service lifecycle (start/stop)        │
└─────────────────────────┬───────────────────────────────────┘
                          │
                          ▼
┌─────────────────────────────────────────────────────────────┐
│                BackgroundEnrichmentService                  │
│  - Priority queue of EnrichmentTasks                        │
│  - Async worker loop                                        │
│  - Retries with backoff                                     │
│  - Updates entities in Neo4j                                │
└─────────────────────────┬───────────────────────────────────┘
                          │
                          ▼
┌─────────────────────────────────────────────────────────────┐
│            CompositeEnrichmentProvider                      │
│  - Tries providers in order until success                   │
│  - Wrapped with CachedEnrichmentProvider                    │
└───────────┬─────────────────────────────────┬───────────────┘
            │                                 │
            ▼                                 ▼
┌───────────────────────┐       ┌─────────────────────────────┐
│  WikimediaProvider    │       │     DiffbotProvider         │
│  - Free, no API key   │       │     - Requires API key      │
│  - Rate: 2 req/sec    │       │     - Structured KG data    │
│  - Wikipedia REST API │       │     - Related entities      │
└───────────────────────┘       └─────────────────────────────┘
----

== Quick Start

=== Enable Enrichment

[source,python]
----
from neo4j_agent_memory import MemorySettings, MemoryClient
from neo4j_agent_memory.config.settings import EnrichmentConfig, EnrichmentProvider

settings = MemorySettings(
    neo4j={"uri": "bolt://localhost:7687", "password": "password"},
    enrichment=EnrichmentConfig(
        enabled=True,
        providers=[EnrichmentProvider.WIKIMEDIA],
    ),
)

async with MemoryClient(settings) as client:
    # Entities are automatically enriched in the background
    entity, _ = await client.long_term.add_entity(
        "Albert Einstein",
        "PERSON",
        confidence=0.9,
    )

    # After a short delay, entity will have enriched data
----

== Configuration

=== EnrichmentConfig

[source,python]
----
from neo4j_agent_memory.config.settings import EnrichmentConfig, EnrichmentProvider

config = EnrichmentConfig(
    # Core settings
    enabled=True,                                    # Enable enrichment
    providers=[EnrichmentProvider.WIKIMEDIA],        # Provider list

    # API keys
    diffbot_api_key="your-key",                      # Diffbot API key

    # Rate limiting
    wikimedia_rate_limit=0.5,                        # Seconds between Wikimedia requests
    diffbot_rate_limit=0.2,                          # Seconds between Diffbot requests

    # Caching
    cache_results=True,                              # Cache enrichment results
    cache_ttl_hours=168,                             # Cache TTL (1 week)

    # Background processing
    background_enabled=True,                         # Enable async processing
    queue_max_size=1000,                             # Max queue size
    max_retries=3,                                   # Retry count for failures
    retry_delay_seconds=60.0,                        # Delay between retries

    # Filtering
    entity_types=["PERSON", "ORGANIZATION", "LOCATION", "EVENT"],  # Types to enrich
    min_confidence=0.7,                              # Minimum confidence threshold

    # API settings
    language="en",                                   # Wikipedia language
    user_agent="neo4j-agent-memory/1.0",             # User-Agent for requests
)
----

== Providers

=== WikimediaProvider

Uses the Wikipedia REST API to fetch entity information. **Free, no API key required.**

[source,python]
----
from neo4j_agent_memory.enrichment import WikimediaProvider

provider = WikimediaProvider(
    rate_limit=0.5,          # 0.5s between requests (2 req/sec max)
    language="en",           # Wikipedia language
    user_agent="my-app/1.0", # User-Agent header
)

result = await provider.enrich("Albert Einstein", "PERSON")

if result.status == EnrichmentStatus.SUCCESS:
    print(f"Description: {result.description}")
    print(f"Summary: {result.summary}")
    print(f"Wikipedia URL: {result.wikipedia_url}")
    print(f"Wikidata ID: {result.wikidata_id}")
    print(f"Image URL: {result.image_url}")
----

**Supported Entity Types:**

* PERSON
* ORGANIZATION
* LOCATION
* EVENT
* OBJECT

**Returned Data:**

[cols="2,3"]
|===
|Field |Description

|`description`
|Short description from Wikipedia
|`summary`
|First paragraph of Wikipedia article
|`wikipedia_url`
|Link to Wikipedia page
|`wikidata_id`
|Wikidata Q identifier (e.g., Q937)
|`image_url`
|Main image from Wikipedia
|===

=== DiffbotProvider

Uses the Diffbot Knowledge Graph API for richer, structured data. **Requires API key.**

[source,python]
----
from neo4j_agent_memory.enrichment import DiffbotProvider

provider = DiffbotProvider(
    api_key="your-api-key",
    rate_limit=0.2,  # 5 req/sec
)

result = await provider.enrich("Apple Inc", "ORGANIZATION")

if result.status == EnrichmentStatus.SUCCESS:
    print(f"Description: {result.description}")
    print(f"Types: {result.metadata.get('types')}")
    print(f"Importance: {result.metadata.get('importance')}")
    print(f"Related: {result.related_entities}")
----

**Supported Entity Types:**

* PERSON
* ORGANIZATION
* LOCATION
* EVENT
* OBJECT (mapped to Product)

**Returned Data:**

[cols="2,3"]
|===
|Field |Description

|`description`
|Entity description from Diffbot KG
|`metadata.types`
|Diffbot entity types
|`metadata.importance`
|Importance score (0-100)
|`related_entities`
|List of related entity names
|`external_urls`
|Links to external sources
|===

== Using Providers Directly

You can use enrichment providers directly without the background service:

[source,python]
----
from neo4j_agent_memory.enrichment import (
    WikimediaProvider,
    DiffbotProvider,
    CachedEnrichmentProvider,
    CompositeEnrichmentProvider,
    EnrichmentStatus,
)

# Single provider
provider = WikimediaProvider()
result = await provider.enrich("Paris", "LOCATION")

# With caching
cached_provider = CachedEnrichmentProvider(
    WikimediaProvider(),
    ttl_hours=24,
)
result = await cached_provider.enrich("Paris", "LOCATION")
# Second call returns cached result
result = await cached_provider.enrich("Paris", "LOCATION")

# Composite: try multiple providers in order
composite = CompositeEnrichmentProvider([
    DiffbotProvider(api_key="..."),  # Try Diffbot first
    WikimediaProvider(),              # Fall back to Wikipedia
])
result = await composite.enrich("Elon Musk", "PERSON")
----

== EnrichmentResult

All providers return an `EnrichmentResult` object:

[source,python]
----
from neo4j_agent_memory.enrichment import EnrichmentResult, EnrichmentStatus

@dataclass
class EnrichmentResult:
    status: EnrichmentStatus           # SUCCESS, NOT_FOUND, ERROR, RATE_LIMITED, SKIPPED
    description: str | None = None     # Short description
    summary: str | None = None         # Longer summary
    metadata: dict[str, Any] = {}      # Provider-specific metadata
    external_urls: list[str] = []      # External links
    image_url: str | None = None       # Primary image
    wikipedia_url: str | None = None   # Wikipedia link
    wikidata_id: str | None = None     # Wikidata identifier
    related_entities: list[str] = []   # Related entity names
    error_message: str | None = None   # Error details if failed
----

**Status Values:**

[cols="1,3"]
|===
|Status |Description

|`SUCCESS`
|Enrichment succeeded
|`NOT_FOUND`
|Entity not found in knowledge base
|`ERROR`
|An error occurred during enrichment
|`RATE_LIMITED`
|Rate limit exceeded (will retry later)
|`SKIPPED`
|Entity type not supported by provider
|===

== Background Service

The `BackgroundEnrichmentService` processes enrichment requests asynchronously:

[source,python]
----
from neo4j_agent_memory.enrichment import BackgroundEnrichmentService

# The service is created and managed by MemoryClient when enrichment is enabled
# You don't typically need to interact with it directly

# Manual usage (advanced):
from neo4j_agent_memory.enrichment import (
    BackgroundEnrichmentService,
    WikimediaProvider,
    EnrichmentTask,
)

service = BackgroundEnrichmentService(
    provider=WikimediaProvider(),
    neo4j_client=client._client,
    max_retries=3,
    retry_delay=60.0,
)

# Start the background worker
await service.start()

# Enqueue an entity for enrichment
await service.enqueue(EnrichmentTask(
    entity_id="entity-uuid",
    entity_name="Albert Einstein",
    entity_type="PERSON",
    priority=1,  # Lower = higher priority
))

# Stop the service gracefully
await service.stop()
----

== Entity Updates

When enrichment completes, the entity in Neo4j is updated with additional properties:

[source,cypher]
----
// After enrichment, entity nodes gain these properties:
(:Entity:Person {
    name: "Albert Einstein",
    type: "PERSON",
    // ... original properties ...

    // Enrichment properties:
    enriched_description: "German-born theoretical physicist...",
    wikipedia_url: "https://en.wikipedia.org/wiki/Albert_Einstein",
    wikidata_id: "Q937",
    image_url: "https://upload.wikimedia.org/...",
    enriched_at: datetime(),
    enrichment_source: "wikimedia"
})
----

== Best Practices

=== Rate Limiting

Both providers enforce rate limits to avoid being blocked:

* **Wikimedia**: Default 0.5s between requests (2 req/sec)
* **Diffbot**: Default 0.2s between requests (5 req/sec)

The background service respects these limits automatically.

=== Caching

Enable caching to avoid redundant API calls:

[source,python]
----
enrichment=EnrichmentConfig(
    cache_results=True,
    cache_ttl_hours=168,  # 1 week
)
----

Cached results are stored in memory. For persistent caching across restarts, you could implement a custom `CachedEnrichmentProvider` with a persistent backend.

=== Entity Type Filtering

Only enrich entity types that benefit from external data:

[source,python]
----
enrichment=EnrichmentConfig(
    entity_types=["PERSON", "ORGANIZATION", "LOCATION"],
    # Skip EVENT and OBJECT if not needed
)
----

=== Confidence Filtering

Only enrich high-confidence entities to avoid wasted API calls:

[source,python]
----
enrichment=EnrichmentConfig(
    min_confidence=0.8,  # Only enrich entities with >= 80% confidence
)
----

=== Graceful Degradation

Enrichment failures don't affect core functionality:

* Entities are stored immediately, regardless of enrichment status
* Failed enrichments are retried with exponential backoff
* After max retries, failures are logged but don't raise exceptions
* The system continues operating even if external services are unavailable

== Creating Custom Providers

Implement the `EnrichmentProvider` protocol:

[source,python]
----
from typing import runtime_checkable, Protocol
from neo4j_agent_memory.enrichment import EnrichmentResult, EnrichmentStatus

@runtime_checkable
class EnrichmentProvider(Protocol):
    """Protocol for entity enrichment providers."""

    @property
    def supported_types(self) -> set[str]:
        """Entity types this provider can enrich."""
        ...

    async def enrich(
        self,
        entity_name: str,
        entity_type: str,
        **kwargs,
    ) -> EnrichmentResult:
        """Enrich an entity with additional data."""
        ...

# Example: Custom provider
class MyCustomProvider:
    @property
    def supported_types(self) -> set[str]:
        return {"PERSON", "ORGANIZATION"}

    async def enrich(
        self,
        entity_name: str,
        entity_type: str,
        **kwargs,
    ) -> EnrichmentResult:
        if entity_type not in self.supported_types:
            return EnrichmentResult(status=EnrichmentStatus.SKIPPED)

        # Fetch data from your source
        data = await self._fetch_from_my_api(entity_name)

        return EnrichmentResult(
            status=EnrichmentStatus.SUCCESS,
            description=data.get("description"),
            metadata={"source": "my-api"},
        )
----

== Troubleshooting

=== Enrichment Not Running

1. Check that enrichment is enabled:
+
[source,python]
----
print(settings.enrichment.enabled)  # Should be True
----

2. Verify the entity type is in the allowed list:
+
[source,python]
----
print(settings.enrichment.entity_types)
----

3. Check entity confidence meets threshold:
+
[source,python]
----
print(settings.enrichment.min_confidence)  # Entity confidence must be >= this
----

=== Rate Limiting Errors

If you see `RATE_LIMITED` status:

* Reduce request rate by increasing `wikimedia_rate_limit` or `diffbot_rate_limit`
* The background service will automatically retry after the delay

=== Diffbot API Errors

* Verify your API key is correct
* Check your Diffbot account has available quota
* Ensure the `DIFFBOT_API_KEY` environment variable is set

=== Missing httpx Dependency

The enrichment providers require `httpx` for HTTP requests:

[source,bash]
----
pip install httpx
# Or install with all extras
pip install neo4j-agent-memory[all]
----
