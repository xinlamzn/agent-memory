= Memory Types
:toc: left
:toclevels: 3
:sectanchors:
:icons: font
:source-highlighter: highlight.js

Neo4j Agent Memory provides three complementary memory types that work together to give AI agents comprehensive memory capabilities.

== Overview

[cols="1,2,2"]
|===
|Memory Type |Purpose |Examples

|**Short-Term**
|What happened and when
|Conversations, messages, experiences

|**Long-Term**
|What the agent knows
|Entities, facts, preferences

|**Reasoning**
|How to solve problems
|Reasoning traces, tool usage patterns
|===

[source]
----
┌─────────────────────────────────────────────────────────────┐
│                      MemoryClient                           │
├───────────────────┬───────────────────┬─────────────────────┤
│                   │                   │                     │
│   SHORT-TERM      │    LONG-TERM      │    REASONING       │
│                   │                   │                     │
│  ┌─────────────┐  │  ┌─────────────┐  │  ┌───────────────┐  │
│  │Conversation │  │  │   Entity    │  │  │ReasoningTrace │  │
│  │   Message   │  │  │  Preference │  │  │ ReasoningStep │  │
│  └─────────────┘  │  │    Fact     │  │  │   ToolCall    │  │
│                   │  └─────────────┘  │  └───────────────┘  │
└───────────────────┴───────────────────┴─────────────────────┘
----

== Short-Term Memory

Short-term memory stores conversation history and experiences - the "what happened" of agent interactions.

=== Core Concepts

**Conversation**:: A session or thread of messages with a unique `session_id`
**Message**:: A single message within a conversation (user or assistant)

=== Adding Messages

[source,python]
----
async with MemoryClient(settings) as memory:
    # Add a user message
    message = await memory.short_term.add_message(
        session_id="user-123-session-1",
        role="user",
        content="What's the weather like in San Francisco?",
        metadata={"source": "web_chat"}
    )

    # Add an assistant response
    await memory.short_term.add_message(
        session_id="user-123-session-1",
        role="assistant",
        content="It's currently 65°F and sunny in San Francisco.",
    )
----

=== Message Options

[source,python]
----
await memory.short_term.add_message(
    session_id="session-id",
    role="user",                      # "user" or "assistant"
    content="Message content",

    # Optional parameters
    conversation_id=None,             # Specific conversation UUID
    extract_entities=True,            # Extract entities (default: True)
    generate_embedding=True,          # Generate embedding (default: True)
    metadata={"key": "value"},        # Custom metadata
)
----

=== Retrieving Messages

[source,python]
----
# Get recent messages from a session
messages = await memory.short_term.get_messages(
    session_id="user-123-session-1",
    limit=20,
    include_metadata=True
)

for msg in messages:
    print(f"[{msg.role}] {msg.content}")
    print(f"  Time: {msg.created_at}")
    print(f"  Metadata: {msg.metadata}")
----

=== Searching Messages

[source,python]
----
# Semantic search across messages
results = await memory.short_term.search_messages(
    query="weather questions",
    session_id="user-123-session-1",  # Optional: filter by session
    limit=10
)

for msg in results:
    score = msg.metadata.get("similarity", 0)
    print(f"[{score:.2f}] {msg.content}")
----

==== Metadata Filters

Search with MongoDB-style metadata filters:

[source,python]
----
# Search with metadata filters
results = await memory.short_term.search_messages(
    query="restaurant",
    session_id="user-123",
    metadata_filters={
        "speaker": "Lenny",                    # Exact match
        "turn_index": {"$gt": 5},              # Greater than
        "source": {"$in": ["web", "mobile"]},  # In list
        "archived": {"$exists": False},        # Field doesn't exist
    },
    limit=10,
)
----

Supported operators:

* `$eq` - Equal (default)
* `$ne` - Not equal
* `$gt`, `$gte` - Greater than (or equal)
* `$lt`, `$lte` - Less than (or equal)
* `$in`, `$nin` - In / not in list
* `$exists` - Field exists / doesn't exist
* `$contains` - String contains (case-insensitive)

# User: What's the weather like in San Francisco?

=== Managing Conversations

[source,python]

# Delete a message

=== Message Linking

Messages in conversations are automatically linked sequentially in the graph for efficient traversal:

[source]
----
(Conversation) -[:FIRST_MESSAGE]-> (Message)     # O(1) access to first message
(Conversation) -[:HAS_MESSAGE]-> (Message)       # Membership check
(Message) -[:NEXT_MESSAGE]-> (Message)           # Sequential chain
----

This linking happens automatically when adding messages individually or in batches. For existing data without links, use the migration utility:

[source,python]
----
# Migrate existing conversations to use message linking
migrated = await memory.short_term.migrate_message_links()
print(f"Migrated {len(migrated)} conversations")
# Returns: {"conversation_id": num_messages_linked, ...}
----

=== Session Management

List and manage sessions with the `list_sessions()` method:

[source,python]
----
# List all sessions with metadata
sessions = await memory.short_term.list_sessions(
    prefix="user-123-",  # Optional: filter by prefix
    limit=50,
    offset=0,
    order_by="updated_at",  # "created_at", "updated_at", or "message_count"
    order_dir="desc",
)

for session in sessions:
    print(f"{session.session_id}: {session.message_count} messages")
    print(f"  First: {session.first_message_preview}")
    print(f"  Last: {session.last_message_preview}")
    print(f"  Created: {session.created_at}")
    print(f"  Updated: {session.updated_at}")
----

=== Batch Loading

Load large amounts of messages efficiently with progress tracking:

[source,python]
----
# Prepare messages for bulk loading
messages = [
    {"role": "user", "content": "Hello!", "metadata": {"source": "web"}},
    {"role": "assistant", "content": "Hi there!", "metadata": {"source": "web"}},
    # ... hundreds more messages
]

# Load with progress callback
def on_progress(loaded, total):
    print(f"Loaded {loaded}/{total} messages")

await memory.short_term.add_messages_batch(
    session_id="bulk-session",
    messages=messages,
    batch_size=100,
    generate_embeddings=True,
    extract_entities=False,  # Defer entity extraction for speed
    on_progress=on_progress,
)

# Generate embeddings later for messages that don't have them
count = await memory.short_term.generate_embeddings_batch(
    session_id="bulk-session",
    batch_size=50,
)
print(f"Generated {count} embeddings")
----

=== Conversation Summaries

Generate summaries of conversations:

[source,python]
----
# Basic summary (no LLM required)
summary = await memory.short_term.get_conversation_summary("user-123")
print(summary.summary)
print(f"Messages: {summary.message_count}")
print(f"Time range: {summary.time_range}")
print(f"Key entities: {summary.key_entities}")
print(f"Key topics: {summary.key_topics}")

# With custom LLM summarizer
async def my_summarizer(transcript: str) -> str:
    response = await openai_client.chat.completions.create(
        model="gpt-4o-mini",
        messages=[
            {"role": "system", "content": "Summarize this conversation concisely."},
            {"role": "user", "content": transcript}
        ]
    )
    return response.choices[0].message.content

summary = await memory.short_term.get_conversation_summary(
    "user-123",
    summarizer=my_summarizer,
    include_entities=True,
)
----

=== Entity Extraction from Session

Extract entities from all messages in a session (batch processing):

[source,python]
----
# Extract entities from messages that don't have them yet
stats = await memory.short_term.extract_entities_from_session(
    session_id="user-123",
    batch_size=50,
    skip_existing=True,  # Skip messages that already have entity links
    on_progress=lambda processed, total: print(f"Processed {processed}/{total}"),
)

print(f"Messages processed: {stats['messages_processed']}")
print(f"Entities extracted: {stats['entities_extracted']}")
----

== Long-Term Memory

Long-term memory stores facts, entities, and preferences - the "what the agent knows" about the world and users.

=== Core Concepts

**Entity**:: A person, place, thing, or concept (using xref:poleo-model.adoc[POLE+O model])
**Preference**:: A user preference or inclination
**Fact**:: A piece of factual information

=== Working with Entities

==== Automatic Extraction

Entities are automatically extracted when adding messages:

[source,python]
----
await memory.short_term.add_message(
    session_id="session-1",
    role="user",
    content="I work at Acme Corp with John Smith in San Francisco"
)
# Automatically extracts:
# - "Acme Corp" (ORGANIZATION)
# - "John Smith" (PERSON)
# - "San Francisco" (LOCATION)
----

==== Manual Entity Creation

[source,python]
----
# Add an entity manually
entity = await memory.long_term.add_entity(
    name="Acme Corporation",
    entity_type="ORGANIZATION",
    subtype="COMPANY",
    description="A technology company",
    aliases=["Acme Corp", "Acme"],
    attributes={"industry": "technology", "founded": 2010},
    generate_embedding=True
)
----

==== Searching Entities

[source,python]
----
# Semantic search
entities = await memory.long_term.search_entities(
    query="technology companies",
    limit=10
)

# Filter by type
people = await memory.long_term.search_entities(
    query="engineers",
    entity_type="PERSON",
    limit=10
)

# Get entity by name
entity = await memory.long_term.get_entity(
    name="Acme Corp",
    entity_type="ORGANIZATION"
)
----

=== Location Geocoding

LOCATION entities can be automatically geocoded to add latitude/longitude coordinates, enabling geospatial queries. See xref:configuration.adoc#_geocoding_configuration[Geocoding Configuration] for setup details.

[source,python]
----
from neo4j_agent_memory import (
    MemoryClient,
    MemorySettings,
    GeocodingConfig,
    GeocodingProvider,
)

# Enable geocoding in settings
settings = MemorySettings(
    geocoding=GeocodingConfig(
        enabled=True,
        provider=GeocodingProvider.NOMINATIM,  # Free, or use GOOGLE for higher accuracy
    )
)

async with MemoryClient(settings) as memory:
    # LOCATION entities are automatically geocoded when added
    entity, _ = await memory.long_term.add_entity(
        name="Empire State Building, New York",
        entity_type="LOCATION",
    )

    # Get coordinates for a location
    coords = await memory.long_term.get_entity_coordinates(entity.id)
    if coords:
        lat, lon = coords
        print(f"Location: {lat}, {lon}")  # 40.7484, -73.9857

    # Find locations near a point
    nearby = await memory.long_term.search_locations_near(
        latitude=40.7484,
        longitude=-73.9857,
        radius_km=5.0,
        limit=10,
    )

    # Find locations within a bounding box
    in_area = await memory.long_term.search_locations_in_bounds(
        min_lat=40.70,
        max_lat=40.80,
        min_lon=-74.02,
        max_lon=-73.95,
    )

    # Batch geocode existing locations without coordinates
    stats = await memory.long_term.geocode_locations(
        batch_size=50,
        skip_existing=True,
    )
    print(f"Geocoded {stats['geocoded']} of {stats['processed']} locations")
----

==== Session-Scoped Location Queries

Location queries can be filtered by conversation session, returning only locations mentioned in a specific conversation. This is useful for building conversation-aware map visualizations.

[source,python]
----
# Get all locations (no session filter)
locations = await memory.get_locations(
    has_coordinates=True,  # Only locations with lat/lon
    limit=500,
)

# Get locations mentioned in a specific conversation
locations = await memory.get_locations(
    session_id="user-123-session-1",
    has_coordinates=True,
    limit=500,
)

for loc in locations:
    print(f"{loc['name']}: ({loc['latitude']}, {loc['longitude']})")
    print(f"  Conversations: {loc['conversations']}")

# Session-scoped geospatial queries
nearby = await memory.long_term.search_locations_near(
    latitude=40.7484,
    longitude=-73.9857,
    radius_km=10.0,
    session_id="user-123-session-1",  # Only locations from this conversation
    limit=50,
)

in_bounds = await memory.long_term.search_locations_in_bounds(
    min_lat=40.70,
    max_lat=40.80,
    min_lon=-74.02,
    max_lon=-73.95,
    session_id="user-123-session-1",  # Only locations from this conversation
)
----

The `get_locations()` method returns location entities with:

* `id` - Entity UUID
* `name` - Location name
* `subtype` - Location subtype (CITY, COUNTRY, REGION, etc.)
* `latitude` / `longitude` - Coordinates (if geocoded)
* `conversations` - List of conversations that mention this location

==== Controlling Geocoding

[source,python]
----
# Add location without geocoding (coordinates provided directly)
entity, _ = await memory.long_term.add_entity(
    name="Custom Location",
    entity_type="LOCATION",
    geocode=False,  # Skip geocoding
    coordinates=(40.7128, -74.0060),  # Provide coordinates directly
)

# Add location without geocoding (to geocode later in batch)
entity, _ = await memory.long_term.add_entity(
    name="Some Location",
    entity_type="LOCATION",
    geocode=False,
)

# Later: batch geocode all locations without coordinates
await memory.long_term.geocode_locations()
----

=== Entity Deduplication

Long-term memory supports automatic entity deduplication when adding entities:

[source,python]
----
from neo4j_agent_memory.memory import DeduplicationConfig

# Configure deduplication thresholds
config = DeduplicationConfig(
    enabled=True,                    # Enable deduplication (default)
    auto_merge_threshold=0.95,       # Auto-merge if similarity >= 0.95
    flag_threshold=0.85,             # Flag for review if >= 0.85 but < 0.95
    use_fuzzy_matching=True,         # Also use fuzzy string matching
    fuzzy_threshold=0.9,             # Fuzzy match threshold
)

# add_entity returns (entity, dedup_result) tuple
entity, dedup_result = await memory.long_term.add_entity(
    name="Jon Smith",
    entity_type="PERSON",
)

# Check what happened
if dedup_result.action == "merged":
    print(f"Auto-merged with {dedup_result.matched_entity_name}")
elif dedup_result.action == "flagged":
    print(f"Flagged as potential duplicate of {dedup_result.matched_entity_name}")
else:
    print("No duplicates found, entity created normally")

# Find and review potential duplicates
duplicates = await memory.long_term.find_potential_duplicates(limit=100)
for entity1, entity2, confidence in duplicates:
    await memory.long_term.review_duplicate(
        source_id=entity1.id,
        target_id=entity2.id,
        confirm=True,  # True to merge, False to reject
    )
----

=== Provenance Tracking

Track where entities were extracted from and which extractor produced them:

[source,python]
----
# Link entity to source message
await memory.long_term.link_entity_to_message(
    entity,
    message_id,
    confidence=0.95,
    start_pos=10,
    end_pos=20,
    context="... mentioned John Smith in the meeting ...",
)

# Link entity to extractor
await memory.long_term.link_entity_to_extractor(
    entity,
    "GLiNEREntityExtractor",
    confidence=0.95,
    extraction_time_ms=150.5,
)

# Get provenance for an entity
provenance = await memory.long_term.get_entity_provenance(entity)
for source in provenance["sources"]:
    print(f"From message {source['message_id']} at position {source['start_pos']}")
for extractor in provenance["extractors"]:
    print(f"Extracted by {extractor['name']} v{extractor['version']}")

# Get all entities extracted from a message
entities = await memory.long_term.get_entities_from_message(message_id)

# Get extraction statistics
stats = await memory.long_term.get_extraction_stats()
print(f"Total entities: {stats['total_entities']}")
print(f"From {stats['source_messages']} messages")
----

== Reasoning Memory

Reasoning memory stores reasoning traces and tool usage patterns - the "how to solve problems" knowledge.

=== Core Concepts

**ReasoningTrace**:: A complete reasoning process for a task
**ReasoningStep**:: A single step in the reasoning process
**ToolCall**:: A tool/function invocation with inputs and outputs

=== Recording Reasoning Traces

[source,python]
----
# Start a reasoning trace (optionally linked to a triggering message)
trace = await memory.reasoning.start_trace(
    task="Find restaurants in San Francisco",
    session_id="session-1",
    triggered_by_message_id=user_message.id,  # Optional: link to message
)

# Add reasoning steps
await memory.reasoning.add_step(
    trace_id=trace.id,
    thought="I need to search for restaurants in the specified location",
    action="search_restaurants",
    observation="Found 10 restaurants"
)

await memory.reasoning.add_step(
    trace_id=trace.id,
    thought="I should filter by the user's dietary preferences",
    action="filter_by_preference",
    observation="3 vegetarian restaurants found"
)

# Complete the trace
await memory.reasoning.complete_trace(
    trace_id=trace.id,
    result="Recommended: Green Garden Cafe",
    success=True
)
----

=== Recording Tool Calls

[source,python]
----
# Record a tool call (optionally linked to a message)
tool_call = await memory.reasoning.record_tool_call(
    step_id=step.id,
    tool_name="search_api",
    arguments={"query": "restaurants", "location": "San Francisco"},
    result={"results": [...]},
    status=ToolCallStatus.SUCCESS,
    duration_ms=150,
    message_id=user_message.id,  # Optional: link to triggering message
)
----

=== Linking Reasoning Memory to Messages

Reasoning traces and tool calls can be linked to the messages that triggered them:

[source]
----
(ReasoningTrace) -[:INITIATED_BY]-> (Message)    # Trace triggered by message
(ToolCall) -[:TRIGGERED_BY]-> (Message)          # Tool call triggered by message
----

This linking can be done at creation time (shown above) or post-hoc:

[source,python]
----
# Link an existing trace to a message
await memory.reasoning.link_trace_to_message(
    trace_id=trace.id,
    message_id=user_message.id
)
----

=== Searching Similar Traces

[source,python]

# Find similar past reasoning
similar_traces = await memory.reasoning.get_similar_traces(

=== Streaming Trace Recording

Use `StreamingTraceRecorder` for recording traces during streaming responses:

[source,python]
----
from neo4j_agent_memory import StreamingTraceRecorder

async with StreamingTraceRecorder(
    memory.reasoning,
    session_id="user-123",
    task="Process customer inquiry"

    # Start a step
    step = await recorder.start_step(
        thought="Analyzing the request",
        action="analyze",

        {"found": 5, "results": [...]},

    # Add observations
    await recorder.add_observation("Found 5 relevant records")

    # Start another step

# Trace is automatically completed with timing when context exits

== Combined Context

The `MemoryClient` provides a unified method to get context from all memory types:

[source,python]
----
async with MemoryClient(settings) as memory:
    # Get combined context
    context = await memory.get_context(
        query="restaurant recommendation",
        session_id="user-123-session-1",

        # Control which memory types to include
        include_short_term=True,    # Recent conversation
        include_long_term=True,     # Facts and preferences
        include_reasoning=True,   # Past reasoning patterns

        max_items=10
    )

    # Use in LLM prompt
    prompt = f"""
    Based on the following context:

    {context}

    Please recommend a restaurant.
    """
----

=== Context Output Format

== Graph Export

Export memory graph data for visualization or external processing:

[source,python]
----
# Export the full memory graph
graph = await memory.get_graph(
    memory_types=["short_term", "long_term", "reasoning"],  # Optional filter
    session_id="user-123",  # Optional session filter
    include_embeddings=False,  # Don't include large embedding vectors
    limit=1000,
)

print(f"Nodes: {len(graph.nodes)}")
print(f"Relationships: {len(graph.relationships)}")

# Access graph data
for node in graph.nodes:
    print(f"{node.labels}: {node.properties.get('name', node.id)}")

for rel in graph.relationships:
    print(f"{rel.from_node} -[{rel.type}]-> {rel.to_node}")
----

The `MemoryGraph` object contains:

* `nodes` - List of `GraphNode` objects with `id`, `labels`, and `properties`
* `relationships` - List of `GraphRelationship` objects with `id`, `type`, `from_node`, `to_node`, and `properties`
* `metadata` - Additional information about the export

== Best Practices

=== Session Management

* Use consistent `session_id` values for related conversations
* Include user identifiers in session IDs for multi-user systems
* Consider session expiration for privacy

=== Entity Extraction

* Enable extraction by default for rich memory
* Disable for bulk loading or sensitive content
* Use appropriate extraction pipeline for your use case

=== Preferences

* Use consistent categories across your application
* Set appropriate importance scores (0.0-1.0)
* Update preferences when user behavior changes

=== Reasoning Memory

* Record both successful and failed reasoning traces
* Include detailed observations for learning
* Use for few-shot prompting with similar tasks
