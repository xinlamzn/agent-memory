== Agent Integration Improvements

[[agent-tool-templates]]
=== Agent Tool Templates ✅ IMPLEMENTED

[horizontal]
Priority:: High
Status:: **DONE** (v0.1.0)
Implementation:: Lennys-Memory demo agent tools

**Features Implemented**:

The Lennys-Memory demo now includes comprehensive agent tools demonstrating how to leverage all memory features:

.Entity Query Tools
* `search_entities` - Semantic search for entities by name, type, or description
* `get_entity_context` - Full entity details with Wikipedia enrichment and podcast mentions
* `find_related_entities` - Find co-occurring entities via graph relationships
* `get_most_mentioned_entities` - Top entities by mention frequency

.Location Query Tools (Map View Integration)
* `search_locations` - Search locations with optional episode filtering
* `find_locations_near` - Geospatial radius search using Haversine distance
* `get_episode_locations` - Geographic profile of a specific episode
* `find_location_path` - Shortest path between locations in the knowledge graph
* `get_location_clusters` - Location density analysis by country for heatmap
* `calculate_location_distances` - Great-circle distance calculation between locations

.Preference and Reasoning Memory Tools
* `get_user_preferences` - Retrieve stored user preferences for personalization
* `find_similar_past_queries` - Find similar past traces for learning from history

.Benefits Achieved
* Agent now uses 80%+ of available memory features (vs ~20% before)
* Entity-based queries work ("Who founded Stripe?")
* Location-based analysis is possible ("What markets does Lenny discuss?")
* Agent learns from past successful reasoning

.Example Tool Implementation
[source,python]
----
@agent.tool
async def tool_search_entities(
    ctx: RunContext[AgentDeps],
    query: str,
    entity_type: str | None = None,
    limit: int = 10,
) -> str:
    """Search for entities mentioned in podcasts."""
    entities = await ctx.deps.client.long_term.search_entities(
        query=query,
        entity_type=entity_type,
        limit=limit,
    )
    return json.dumps([{
        "name": e.name,
        "type": e.type,
        "description": e.description,
        "wikipedia_url": e.wikipedia_url,
    } for e in entities])
----

@agent.system_prompt
async def add_memory_context(ctx: RunContext[AgentDeps]) -> str:
    """Add preferences and traces to system prompt."""
    parts = []

    # Get user preferences
    preferences = await ctx.deps.client.long_term.search_preferences("", limit=10)
    if preferences:
        pref_list = [f"- {p.category}: {p.preference}" for p in preferences]

    # Get similar past traces
    if ctx.deps.current_query:
        traces = await ctx.deps.client.reasoning.get_similar_traces(
            task=ctx.deps.current_query,
            limit=2,
            success_only=True,
        )
        if traces:
            trace_items = [f"- Query: {t.task}, Outcome: {t.outcome}" for t in traces]
            parts.append(f"## Past Interactions\n{chr(10).join(trace_items)}")

    return "\n\n".join(parts)

**Features Implemented**:

.Benefits Achieved
* Zero API costs (no LLM calls)
* Fast extraction (model-based, not LLM)
* Full offline support
* Customizable relation types

.Example Usage
[source,python]
----

    GLiNERWithRelationsExtractor,
    is_glirel_available,
)

if is_glirel_available():
    extractor = GLiNERWithRelationsExtractor.for_poleo()
    result = await extractor.extract("John works at Acme Corp in NYC.")
    print(result.entities)   # John, Acme Corp, NYC
    print(result.relations)  # John -[WORKS_AT]-> Acme Corp
----

[[custom-schema-persistence]]
=== Custom Schema Persistence ✅ IMPLEMENTED

[horizontal]
Priority:: Medium
Status:: **DONE** (v0.1.0)
Implementation:: `SchemaManager` class in `neo4j_agent_memory.schema.persistence`

**Features Implemented**:

* `SchemaManager` class for storing and loading schemas from Neo4j
* `StoredSchema` dataclass with full metadata (id, name, version, description, config, is_active, created_at, created_by)
* `SchemaListItem` dataclass for listing schemas with summary info
* Schema versioning support: multiple versions of the same schema can coexist
* Active version tracking: one version per schema name is marked as active
* Methods:
  - `save_schema()` - Save schema to Neo4j with optional `created_by` and `set_active` parameters
  - `load_schema()` - Load active schema by name
  - `load_schema_version()` - Load specific version
  - `list_schemas()` - List all schemas with version counts
  - `list_schema_versions()` - List all versions of a schema
  - `set_active_version()` - Set specific version as active
  - `delete_schema()` - Delete schema by ID
  - `delete_all_versions()` - Delete all versions of a schema
  - `schema_exists()` - Check if schema exists
  - `ensure_schema_index()` - Create indexes for efficient lookups
* Schemas stored as `(:Schema)` nodes with JSON-serialized `EntitySchemaConfig`

.Benefits Achieved
* No code changes required for new domains
* Schema versioning and audit trails
* Multi-tenant schema management

.Example Usage
[source,python]
----
from neo4j_agent_memory.schema import SchemaManager, EntitySchemaConfig

manager = SchemaManager(client)

# Save schema
schema = EntitySchemaConfig(name="medical", version="1.0", ...)
stored = await manager.save_schema(schema, created_by="admin")

# Load schema
loaded = await manager.load_schema("medical")

# List all schemas
schemas = await manager.list_schemas()
for s in schemas:
    print(f"{s.name}: v{s.latest_version} ({s.version_count} versions)")

# Version management
await manager.set_active_version("medical", "2.0")
----

.Neo4j Schema Node
[source,cypher]
----
(:Schema {
    id: "uuid",
    name: "medical",
    version: "1.0",
    description: "Medical records schema",
    config: "{...}",  // JSON-serialized EntitySchemaConfig
    is_active: true,
    created_at: datetime(),
    created_by: "admin"
})
----

== Pipeline and Performance Improvements

[[streaming-extraction]]
=== Streaming Extraction for Long Documents ✅ IMPLEMENTED

[horizontal]
Priority:: Medium
Status:: **DONE** (v0.1.0)
Implementation:: `StreamingExtractor` class in `neo4j_agent_memory.extraction.streaming`

**Features Implemented**:

* `StreamingExtractor` class wrapping any `EntityExtractor` for chunked processing
* `extract_streaming()` async generator that yields `StreamingChunkResult` for each chunk
* `extract()` method for complete extraction with automatic deduplication
* `extract_to_result()` convenience method returning standard `ExtractionResult`
* Chunking strategies:
  - `chunk_text_by_chars()` - Character-based chunking with sentence boundary detection
  - `chunk_text_by_tokens()` - Token-based chunking (approximate word count)
* Configurable overlap between chunks to prevent entity splitting
* Automatic entity position adjustment to document-level coordinates
* Entity and relation deduplication across chunks (by name+type, keeping highest confidence)
* Progress callback support for UI integration
* `StreamingExtractionStats` with detailed metrics (total/successful/failed chunks, raw/deduplicated entity counts)
* `ChunkInfo` dataclass with chunk metadata (index, start/end positions, first/last flags)
* `create_streaming_extractor()` factory function with sensible defaults

.Benefits Achieved
* Handle arbitrarily long documents (>100K tokens)
* Lower memory footprint (process one chunk at a time)
* Real-time progress updates via async generator
* Automatic deduplication across chunk boundaries

.Example Usage
[source,python]
----
from neo4j_agent_memory.extraction import (
    StreamingExtractor,
    create_streaming_extractor,
    GLiNEREntityExtractor,
)

extractor = GLiNEREntityExtractor.for_schema("podcast")
streamer = StreamingExtractor(extractor, chunk_size=4000, overlap=200)

# Stream results chunk by chunk
async for chunk_result in streamer.extract_streaming(long_document):
    print(f"Chunk {chunk_result.chunk.index}: {chunk_result.entity_count} entities")

# Or get complete result with deduplication
result = await streamer.extract(long_document, deduplicate=True)
print(f"{result.stats.deduplicated_entities} unique entities from {result.stats.total_chunks} chunks")
----

.Chunking Options
[source,python]
----
# Character-based (default)
streamer = StreamingExtractor(extractor, chunk_size=4000, overlap=200)

# Token-based (approximate)
streamer = StreamingExtractor(extractor, chunk_size=1000, overlap=50, chunk_by_tokens=True)

# Factory with defaults
streamer = create_streaming_extractor(extractor)
----

[horizontal]

Implementation:: `ExtractionPipeline.extract_batch()` and `GLiNEREntityExtractor.extract_batch()`

* `BatchExtractionResult` dataclass with aggregate statistics (total_items, successful_items, failed_items, success_rate, total_entities, total_relations)

* Configurable batch size via `batch_size` parameter for memory management

.Example Usage
[source,python]
----

    texts,
    batch_size=10,
    max_concurrency=5,
    on_progress=lambda done, total: print(f"{done}/{total}"),
)
print(f"Extracted {result.total_entities} entities from {result.successful_items} texts")

== Neo4j Storage Improvements

[[entity-deduplication]]
=== Entity Deduplication on Ingest ✅ IMPLEMENTED

[horizontal]
Priority:: High
Status:: **DONE** (v0.1.0)
Implementation:: `LongTermMemory.add_entity()` with `DeduplicationConfig`

**Features Implemented**:

* `DeduplicationConfig` dataclass for configuring deduplication thresholds and behavior
* `DeduplicationResult` dataclass returned with every `add_entity()` call indicating action taken
* Embedding-based similarity search to find potential duplicates
* Optional fuzzy string matching (via rapidfuzz) to boost confidence scores
* Configurable thresholds:
  - `auto_merge_threshold` (default 0.95): Automatically merge entities above this similarity
  - `flag_threshold` (default 0.85): Flag entities between this and auto_merge for human review
* `SAME_AS` relationship creation for flagged duplicates with status tracking (`pending`, `confirmed`, `rejected`)
* `match_same_type_only` option to only match entities of the same type
* `find_potential_duplicates()` method to retrieve pending duplicate reviews
* `review_duplicate()` method to confirm or reject duplicate pairs
* `merge_duplicate_entities()` method to manually merge entities
* `get_same_as_cluster()` method to get all entities linked via SAME_AS
* `get_deduplication_stats()` method for statistics
* `deduplicate=False` parameter on `add_entity()` to skip deduplication for specific entities

.Example Usage
[source,python]
----
from neo4j_agent_memory.memory import LongTermMemory, DeduplicationConfig

config = DeduplicationConfig(
    auto_merge_threshold=0.95,
    flag_threshold=0.85,
    use_fuzzy_matching=True,
)

memory = LongTermMemory(client, embedder, deduplication=config)

# add_entity returns (entity, dedup_result) tuple
entity, result = await memory.add_entity("Jon Smith", "PERSON")
if result.action == "merged":
    print(f"Auto-merged with {result.matched_entity_name}")
elif result.action == "flagged":
    print(f"Flagged for review against {result.matched_entity_name}")

# Review pending duplicates
duplicates = await memory.find_potential_duplicates()
for e1, e2, confidence in duplicates:
    await memory.review_duplicate(e1.id, e2.id, confirm=True)
----

== Integration Improvements

[[opentelemetry]]
=== OpenTelemetry and Opik Tracing ✅ IMPLEMENTED

[horizontal]
Priority:: Medium
Status:: **DONE** (v0.1.0)
Implementation:: `neo4j_agent_memory.observability` module

**Features Implemented**:

* Abstract `Tracer` and `Span` interfaces for provider-agnostic tracing
* `OpenTelemetryTracer` with OTLP export support
* `OpikTracer` for LLM-focused observability (Comet's Opik platform)
* `NoOpTracer` for zero-overhead disabled tracing
* `get_tracer()` function with auto-detection (Opik > OpenTelemetry > NoOp)
* Decorator-based tracing: `@tracer.trace("function_name")`
* Context manager support: `with tracer.span("name") as span:`
* Async support: `async with tracer.async_span("name") as span:`
* Span attributes, status, and exception recording
* Duration tracking for performance monitoring

.Installation
[source,bash]
----
# OpenTelemetry support
uv pip install neo4j-agent-memory[opentelemetry]

# Opik support
uv pip install neo4j-agent-memory[opik]
----

.Example Usage
[source,python]
----
from neo4j_agent_memory.observability import get_tracer

# Auto-detect available provider
tracer = get_tracer()

# Or specify explicitly
tracer = get_tracer(provider="opentelemetry", service_name="my-service")
tracer = get_tracer(provider="opik", project_name="my-project")

# Decorator-based tracing
@tracer.trace("extract_entities")
async def extract(text: str):
    result = await extractor.extract(text)
    return result

# Context manager for manual spans
async with tracer.async_span("extraction") as span:
    span.set_attribute("text_length", len(text))
    result = await extract(text)
    span.set_attribute("entity_count", len(result.entities))
----

.Benefits Achieved
* Performance monitoring with spans and timing
* Provider flexibility (OpenTelemetry, Opik, or disabled)
* Zero overhead when tracing disabled
* LLM-specific observability via Opik

[[benchmarks]]
=== Extraction Quality Benchmarks ✅ IMPLEMENTED

[horizontal]
Priority:: Medium
Status:: **DONE** (v0.1.0)
Implementation:: `benchmarks/` module with metrics and runner

**Features Implemented**:

* `benchmarks/metrics.py` - Precision, recall, F1 calculation utilities:
  - `EntityMetrics` - Per-entity-type metrics (TP, FP, FN, precision, recall, F1)
  - `ExtractionMetrics` - Aggregate metrics with micro/macro averages
  - `ExpectedEntity` - Test case entity with alias matching support
  - `BenchmarkResult` - Complete benchmark run results
  - `calculate_entity_metrics()` and `calculate_extraction_metrics()` functions
* `benchmarks/runner.py` - Benchmark execution framework:
  - `BenchmarkTestCase` - Single test case with text and expected entities
  - `BenchmarkConfig` - Configuration (warmup_runs, num_runs, timeout)
  - `BenchmarkSuite` - Collection of test cases with JSON serialization
  - `BenchmarkRunner` - Async runner for executing benchmarks
  - `compare_extractors()` - Run same suite across multiple extractors
  - `create_sample_benchmark_suite()` - Sample test suite for getting started
* JSON file support for loading/saving benchmark suites
* Latency and throughput measurements
* Error handling with detailed error collection

.Metrics Tracked
* Precision/Recall/F1 by entity type
* Micro-averaged metrics (aggregate TP/FP/FN)
* Macro-averaged metrics (average of per-type scores)
* Latency (total, average per document)
* Throughput (documents per second)

.Example Usage
[source,python]
----
from benchmarks import (
    BenchmarkRunner,
    BenchmarkSuite,
    BenchmarkTestCase,
    ExpectedEntity,
    create_sample_benchmark_suite,
)
from neo4j_agent_memory.extraction import GLiNEREntityExtractor

# Create extractor and runner
extractor = GLiNEREntityExtractor.for_schema("poleo")
runner = BenchmarkRunner(extractor)

# Run sample suite
suite = create_sample_benchmark_suite()
result = await runner.run_suite(suite)

# Access metrics
print(f"Micro F1: {result.metrics.micro_f1:.3f}")
print(f"Macro F1: {result.metrics.macro_f1:.3f}")
print(f"Avg latency: {result.avg_latency_ms:.1f}ms")

# Per-entity-type metrics
for entity_type, metrics in result.metrics.entity_metrics.items():
    print(f"{entity_type}: P={metrics.precision:.2f}, R={metrics.recall:.2f}")

# Compare extractors
results = await runner.compare_extractors([extractor1, extractor2], suite)
----

.Creating Custom Benchmark Suites
[source,python]
----
suite = BenchmarkSuite(
    name="custom_benchmark",
    description="Domain-specific benchmark",
    test_cases=[
        BenchmarkTestCase(
            id="tc-001",
            text="John Smith works at Acme Corp.",
            expected_entities=[
                ExpectedEntity(name="John Smith", entity_type="PERSON"),
                ExpectedEntity(
                    name="Acme Corp",
                    entity_type="ORGANIZATION",
                    aliases=["Acme Corporation", "Acme"],
                ),
            ],
        ),
    ],
)

# Save/load from JSON
suite.to_json_file("my_benchmark.json")
loaded = BenchmarkSuite.from_json_file("my_benchmark.json")
----
